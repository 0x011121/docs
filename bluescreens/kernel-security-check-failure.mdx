# KERNEL\_SECURITY\_CHECK

This Blue Screen of Death (BSOD) can be triggered by a wide range of system conflicts or security settings. Please follow this guide carefully to resolve the issue.

## Common Cause: Outdated Version

Before changing your system settings, you must verify the status of the cheat.

<Warning>
  **Is the cheat updated?**

  A common cause for this specific crash is attempting to inject a cheat that is **outdated**.

  Please check the `#announcements` channel in our Discord. If the cheat is currently marked as **UPDATING**, this crash is expected behavior. **Do not** attempt to inject until a new update is released.
</Warning>

## System Configuration

If the cheat is up to date and you are still crashing, you must disable specific Windows security features that block the driver.

<Steps>
  <Step title="Disable Core Isolation">
    1. Open the Windows Start menu and search for **"Core Isolation"**.
    2. Locate the **Memory Integrity** toggle.
    3. Set it to **OFF**.
  </Step>
  <Step title="Disable Exploit Protection">
    1. Search for **"Exploit Protection"** in the Windows Start menu.
    2. You will see a list of settings (Control Flow Guard, Data Execution Prevention, etc.).
    3. Ensure **ALL** options in this list are set to **OFF (By Default)**.
  </Step>
  <Step title="Disable Vulnerable Driver Blocklist">
    This feature actively blocks third-party drivers.

    1. Open **Windows Security**.
    2. Navigate to **Device Security** \> **Core Isolation details**.
    3. Find **Microsoft Vulnerable Driver Blocklist** and ensure it is **OFF**.

    _(Note: If you do not see this toggle, it may already be disabled via Registry or Group Policy)._
  </Step>
</Steps>

<Note>
  **Restart Required:** After changing any of the settings above, you must restart your computer for the changes to take effect.
</Note>
